BONJOUR_CONFIG

 

Description

Get or Set bonjour Configurations.

The device could assign the IP address (link local IP address) by itself through bonjour protocol. The management device could discover the device and configure the device without any change of its networking settings like IP, netmask and gateway addresses. The term zero-configuration is implemented by this protocol.

The friendly name could be the networking alias name to identify the device. If friendly name is an empty string, the default friendly name will be applied automatically when the bonjour is enabled. The mechanism of creating default friendly name is the same as the default friendly name in uPnP. This mechanism is not addressed in the URL command design specification.

Input Argument

state,name
where
state: 0 (disable) or 1 (enabled).
name: Bonjour friendly name in 63 bytes character string. These characters should be readable and the space ,' ', character is acceptable. If state = 0, this argument could be omitted. if state = 1 and the name argument is missing, the device will use its current Bonjour Friendly name setting to start its Bonjour service.

Attribute

  • Read and Write>
  • On-Fly

Return of Message

  • Read Operation
BONJOUR_CONFIG='0'
or
BONJOUR_CONFIG='1,IPCAM-00001'
  • Write Operation
OK: BONJOUR_CONFIG='0'
or
OK: BONJOUR_CONFIG='1,IPCAM-00001'
or
ERROR: BONJOUR_CONFIG='0'

Applicability

  • PlatformW: All devices
  • PlatformA: All devices
  • PlatformT: All devices

Status

Valid

Example

  • Read Operation
http://ip:port/cgi-bin/system?USER=admin&PWD=123456&BONJOUR_CONFIG
or
http://ip:port/cgi-bin/cmd/system?BONJOUR_CONFIG
  • Write Operation
http://ip:port/cgi-bin/system?USER=admin&PWD=123456&BONJOUR_CONFIG=1,IPCAM-00001
or
http://ip:port/cgi-bin/cmd/system?BONJOUR_CONFIG=1,IPCAM-00001
