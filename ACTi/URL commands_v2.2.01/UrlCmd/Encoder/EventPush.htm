EVENT_PUSH

 

Description

Trigger an event manually.
People could force the encoder to response the event by using this URL. The event rule has to be set before manual event is triggered.

The manual trigger could be used to verify if the event rule is configured correctly. The event will be responded no matter the event is in the effective time period or the real trigger source is active. The user could check if the event output meets the expectation immediately by using this manual trigger. They don't need to wait for the real trigger source is active. That helps to save the setup time.

Another application is used when the event source is not from the encoder itself. The external device could send this URL to ask encoder to do something for it.

The encoder will extend the action time of the event which the manual trigger is associated with as the manual trigger occurs.
The encoder will ignore the manual trigger if the previous manual trigger is not returned. The error message "ERROR: internal error. device is busy on previous EVENT_PUSH" will be returned.

Input Argument

index
where

Attribute

  • Write-Only
  • On-Fly

Return of Message

OK
or
ERROR: invalid parameters (if the index is out of range)
or
ERROR: EVENT_PUSH is write-only (if the index is missing)
or
ERROR: internal error. device is busy on previous EVENT_PUSH
or
ERROR: internal error. error description.

Applicability

  • PlatformA: All Devices (Encoder:3.13.xx and later)
  • PlatformT: All Devices (4.09.xx and later)
  • Multi-Channel Video Server: ACD2200, ACD2300 and ACD2400

Status

Valid

See Also

EVENT_CONFIG

Example

http://ip:port/cgi-bin/encoder?USER=Admin&PWD=123456&EVENT_PUSH=1
or
http://ip:port/cgi-bin/cmd/encoder?EVENT_PUSH=1
