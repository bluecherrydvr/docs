VIDEO_STREAM

Description

Get or Set the type of the video streaming mode.

This URL configures the video streaming mode for some specific encoder devices.

In dual streams encoder devices, the available streaming mode is SINGLE or DUAL. In SINGLE streaming mode, the device is able to encode a single video stream. In DUAL streaming mode, the device could encode a video source to two different resolution and frame number video streams.

In QUAD video servers, the available streaming mode is SINGLE, SEQUENTIAL, and QUAD. in SINGLE streaming mode, the device have 4 video streams from 4 video sources. These 4 video streams shares the encoding bandwidth in the device. For example, every video stream has 7.5 PFS in D1. The device could stream one of them to remote device by request. In SEQUENTIAL mode, it is similar to SINGLE mode, the device will switch the video stream in channel 1, 2, 3 and 4 order with the configurable displaying time. The frame rate in each video stream is 7.5 frames/second. In QUAD streaming mode, the device have a single video stream which encodes 4 video sources. These 4 video sources was encoded together into a single video frame in D1 size (every video source is encoded in CIF size).

In 1CH encoder or Multi-Channel devices, only SINGLE streaming mode is acceptable. This is for cross-platform consideration. The error message will be returned if it sets not SINGLE streaming mode.

Input Argument

SINGLE, DUAL, SEQUENTIAL, QUAD

Possible Input Argument with platform devices

Platfrom Devices Possible Input Argument Attribute
1CH or Multi-Channel Encoder SINGLE RW
PlatfromT Encoder SINGLE, DUAL RW
EPL QUAD Video Server SINGLE, SEQUENTIAL, QUAD RW

Attribute

  • Readable and Writable
  • On-Fly

Return of Message

  • Read Operation
VIDEO_STREAM='SINGLE'
  • Write Operation
OK: VIDEO_STREAM='SINGLE'
or
ERROR: VIDEO_STREAM='SINGLE'

Applicability

  • WPL Platform: All Encoders
  • EPL Platform : All Encoders
  • PlatfotmT : All Encoders
  • Multi-Channel Encoder : All Encoders

Status

Valid

Example

  • Read Operation

http://ip:port/cgi-bin/encoder?USER=Admin&PWD=123456&VIDEO_STREAM
or
http://ip:port/cgi-bin/cmd/encoder?VIDEO_STREAM

  • Write Operation
http://ip:port/cgi-bin/encoder?USER=Admin&PWD=123456&VIDEO_STREAM=SINGLE
or
http://ip:port/cgi-bin/cmd/encoder?VIDEO_STREAM=SINGLE
