VIDEO_FPS_CAP

Description

Get the video FPS number capabilities.

Because the TV Standard, NTSC and PAL, in D1 encoder could not be exist at the same time, the video FPS capabilities will be given based on the current TV standard in the device.

The format of the return is CFPS1,CFPS2,CFPS2,CFPS3,...-VFPS1,VFPS2,VFPS3,... where CPSn is the is the FPS number in FPS Mode=MODE1 (Constant FPS Mode) and VFPSn is the FPS number in FPS Mode=Mode2. The '-' is used to separate these two FPS mode. The FPS1 is the smallest FPS number and the FPS2 is larger. There is no space characters in the return message.

Because there is no specific FPS number in the dual streams encoders, the return of this URL is NONE.

For example,

In the  EPL D1 NTSC camera, the return of this URL could be VIDEO_FPS_CAP='1,2,3,4,5,6,7,10,15,30-1,2,3,5,7,15,30'

In the dual streams camera, the return of this URL could be VIDEO_FPS_CAP='NONE'

If the CHANNEL=n was not set with this URL to the Multi-Channel device, the CHANNLE 1 video server's VIDEO_FPS_CAP configurations will be returned. If the CHANNEL 1 video server does not connect to the router, the error message will be returned.

Input Argument

None

Attribute

  • Read Only

Return of Message

VIDEO_FPS_CAP='1,2,3,4,5,6,7,10,15,30-1,2,3,5,7,15,30'

Applicability

  • WPL Platform : All Encoder Devices
  • EPL Platform : All Encoder Devices
  • PlatfotmT : All Encoder Devices
  • Multi-Channel Encoder : All Devices

Status

Valid
